# Провайдеры и сервис-провайдеры

## Зачем нужен модуль управления провайдерами?

Модуль Vendors помогает централизованно управлять всеми внешними провайдерами и сервис-партнерами семейного офиса. Это критически важно для:

- Контроля качества услуг через SLA и scorecards
- Мониторинга расходов и верификации счетов
- Управления рисками третьих сторон
- Обеспечения compliance и безопасности
- Планирования продлений контрактов

## Основные возможности

### Реестр провайдеров (Vendors)

- Централизованная база всех провайдеров
- Типы: банки, брокеры, аудиторы, юристы, налоговые консультанты, кастодианы, IT
- Контактная информация и регионы работы
- Оценка рисков (low/medium/high)
- Онбординг статус (KYC/KYB, security questionnaire)

### Контракты (Contracts)

- Управление договорами с провайдерами
- Трекинг дат начала, окончания, продления
- Fee модели: fixed, AUM-based, hourly, transaction, retainer, hybrid
- Автоматические напоминания о продлении
- Привязка документов из Document Vault
- Анализ условий контракта (termination clause, notice period)

### SLA мониторинг

- Определение KPI для каждого сервиса
- Автоматический расчет метрик
- Статусы: OK, Warning, Breached
- История нарушений
- Автоматические уведомления при breach
- Связь с инцидентами

### Scorecards (оценки качества)

- Периодическая оценка провайдеров
- Критерии оценки:
  - Responsiveness (отзывчивость)
  - Accuracy (точность)
  - Security Posture (безопасность)
  - Cost Efficiency (эффективность затрат)
  - Partnership (партнерство)
- Взвешенная общая оценка
- Тренды: improving, stable, declining
- Рекомендации по улучшению

### Инциденты

- Фиксация проблем с провайдерами
- Severity: low, medium, high, critical
- Категории: service_outage, data_breach, sla_breach, billing_error, etc.
- Timeline событий
- Root cause analysis
- Preventive measures
- Связь с Cases (модуль 36)
- Связь с Security Incidents (модуль 17)

### Счета и верификация комиссий

- Трекинг всех счетов от провайдеров
- Привязка к контрактам
- Автоматическое сравнение с ожидаемыми суммами
- Anomaly detection (обнаружение аномалий)
- Флаги: превышение лимита, нестандартная сумма
- Связь с модулем Fee & Billing (модуль 13)

### Доступы и шаринг (Access)

- Управление доступами через Consents (модуль 34)
- Предоставление и отзыв доступа
- Scope ограничения
- Expiration dates
- Audit trail всех изменений

## AI-ассистент

Модуль включает AI-функции для:

- **Summarize Contract**: краткое изложение ключевых условий контракта
- **Detect Fee Anomalies**: анализ счетов и выявление аномалий
- **Draft Renewal Memo**: подготовка меморандума для продления контракта

Все AI-результаты включают:
- Уровень уверенности (confidence)
- Список допущений (assumptions)
- Источники данных (sources)

## Интеграции

- **Documents (модуль 10)**: хранение контрактов и сертификатов
- **Consents (модуль 34)**: управление доступами
- **Cases (модуль 36)**: связь инцидентов с кейсами
- **Fee & Billing (модуль 13)**: верификация счетов
- **KYC/KYB (модуль 8)**: онбординг провайдеров
- **Security Center (модуль 17)**: security questionnaire, инциденты
- **Notifications (модуль 35)**: уведомления о продлениях и нарушениях

## RBAC (роли и доступы)

- **Admin/Operations**: полное управление провайдерами
- **Compliance**: просмотр security posture и инцидентов
- **CFO**: счета и верификация комиссий
- **Relationship Manager**: доступы и коммуникации
- **External Advisor**: ограниченный просмотр через consents

## Audit Trail

Все действия логируются:
- Создание/обновление провайдера
- Создание/обновление контракта
- Изменение SLA, breach events
- Создание scorecard
- Создание/закрытие инцидента
- Флаг аномалии на счете
- Предоставление/отзыв доступа

## Дисклеймер

**Важно**: Информация о провайдерах носит демонстрационный характер. Все контракты и юридические документы требуют профессиональной юридической проверки перед подписанием или использованием.
