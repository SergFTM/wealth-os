# Эффективность портфеля (Performance)

## Что такое Performance в MFO

Модуль «Эффективность портфеля» отвечает за измерение, анализ и объяснение доходности инвестиционного портфеля. Стандарты измерения соответствуют GIPS (Global Investment Performance Standards).

## TWR vs MWR

### TWR (Time-Weighted Return)
- Исключает влияние денежных потоков
- Показывает «чистую» эффективность управления
- Используйте для оценки менеджера/стратегии

### MWR (Money-Weighted Return)
- Учитывает размер и время денежных потоков
- Показывает реальную доходность для инвестора
- Используйте для персонального отчета клиенту

## Денежные потоки

При расчете доходности учитываются:
- **Inflow** — пополнение портфеля
- **Outflow** — вывод средств
- **Dividend** — полученные дивиденды
- **Fee** — комиссии и расходы

Каждый flow имеет дату, источник и связь с документом.

## Бенчмарки

Поддерживаемые типы бенчмарков:
- **Индексы** — MSCI ACWI, S&P 500, Bloomberg Agg
- **ETF** — SPY, VTI, AGG
- **Кастомные** — взвешенные комбинации

Сравнение показывает:
- Excess return (alpha)
- Tracking error
- Information ratio (MVP light)

## Атрибуция

Разложение доходности по измерениям:
- **Класс актива** — акции, облигации, альтернативы
- **Стратегия** — growth, value, income
- **География** — US, Europe, EM, Asia
- **Ликвидность** — liquid, illiquid
- **Консультант** — вклад каждого advisor

Для каждого сегмента показываем:
- Weight (средний вес за период)
- Return (доходность сегмента)
- Contribution (вклад в общую доходность)
- Flow impact (влияние flows)
- Fee impact (влияние комиссий)

## Консультанты (Advisors)

Модуль отслеживает эффективность внешних консультантов:
- AUM под управлением
- Доходность за период
- Excess vs бенчмарк
- Мандат и стратегия

## Источники данных (Source-First)

Все данные имеют источник:
- **Custodian** — выписки депозитария
- **GL** — главная книга
- **Pricing** — маркет-дата провайдер
- **Manual** — ручной ввод

As-of дата фиксирует актуальность данных.

## Audit-Ready

Все изменения логируются:
- Создание/изменение view
- Генерация отчета
- Изменение бенчмарка
- Добавление flow

Версионность отчетов позволяет сравнивать исторические срезы.

## Типовые сценарии

### «Почему портфель просел?»
1. Откройте атрибуцию за период
2. Найдите сегменты с негативным contribution
3. Нажмите «Объяснить» для AI-драфта

### «Кто дал вклад в рост?»
1. Посмотрите раздел «Влияние консультантов»
2. Сравните excess return по advisors
3. Откройте детали мандата

### «Как сравнить с ACWI?»
1. В настройках view выберите бенчмарк ACWI
2. На дашборде появится excess return
3. Таблица покажет сравнение по периодам

## RBAC и Client-Safe

### Client-Safe режим
- Видны только опубликованные views
- Скрыты внутренние notes и flows источники
- Скрыты детали по консультантам
- Документы только со статусом Shared

### Полный доступ (Owner/Admin/CIO)
- Все views и черновики
- Детальная атрибуция
- Manage benchmarks
- Export и share

### External Advisor
- Только назначенные портфели
- Read-only режим
- Может добавлять draft комментарии
