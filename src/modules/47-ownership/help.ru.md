# Ownership Map — Справка

## Обзор

Модуль **Ownership Map** визуализирует структуру владения семьи:
- **Household** → **Trusts** → **Entities** → **Partnerships/SPV** → **Accounts** → **Assets**

## Основные понятия

### Nodes (Узлы)

Узлы представляют элементы структуры владения:

| Тип | Описание |
|-----|----------|
| Household | Домохозяйство — корневой узел семьи |
| Trust | Траст — структура доверительного управления |
| Entity | Юридическое лицо (LLC, Corp, etc.) |
| Partnership | Партнерство (LP, LLP, etc.) |
| SPV | Special Purpose Vehicle |
| Account | Счет в банке/кастодиане |
| Asset | Конечный актив |

### Links (Связи)

Связи определяют отношения владения между узлами:

- **ownershipPct** — процент владения (0-100%)
- **profitSharePct** — процент участия в прибыли (может отличаться от владения)
- **effectiveFrom** / **effectiveTo** — период действия связи

### UBO (Ultimate Beneficial Owner)

Конечный бенефициар — физическое лицо, которое:
- Прямо или косвенно владеет 25%+ сущности
- Или контролирует сущность иным способом

Расчет UBO:
1. Находим все пути от бенефициара до целевой сущности
2. Умножаем доли владения вдоль каждого пути
3. Суммируем результаты по всем путям

## Функции

### Карта владения (/m/ownership/map)

Интерактивная визуализация графа владения:
- Клик на узел открывает детали в правой панели
- Клик на связь показывает параметры связи
- Zoom/Pan для навигации

### Расчет UBO

Автоматический расчет конечных бенефициаров:
- Учитывает все пути владения
- Порог 25% для квалификации как UBO
- Сохраняет историю расчетов

### Концентрации

Анализ концентраций риска:
- Entity concentration — зависимость от одной сущности
- Jurisdiction concentration — концентрация в одной юрисдикции
- Single point of failure — критические точки в структуре

### Client-Safe View

Публикация для клиентского портала:
- Маскирование номеров счетов
- Исключение внутренних заметок
- Опциональное округление процентов

## Источники

Каждая связь должна иметь источник:
- Документ из хранилища
- Описание источника
- Дата "по состоянию на"

## Audit Trail

Все изменения логируются:
- Создание/обновление/удаление узлов
- Создание/обновление/удаление связей
- Запуски расчета UBO
- Публикации client-safe view

## Дисклеймер

> **Важно:** Ownership карта демонстрационная. Для юридически значимых выводов требуется проверка документов и юристов.
