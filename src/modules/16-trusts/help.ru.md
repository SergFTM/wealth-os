# Трасты и наследование

## Обзор модуля

Модуль **Трасты и наследование** предназначен для управления трастовыми структурами, фондами и завещательными механизмами в рамках Family Office.

> **Важно:** Информация в данном модуле не является юридической консультацией. Для принятия решений обратитесь к квалифицированному юристу.

## Основные понятия

### Что такое траст?

**Траст (Trust)** — правовая структура, в которой:
- **Учредитель (Settlor)** передаёт активы
- **Доверительный управляющий (Trustee)** управляет активами
- **Бенефициары (Beneficiaries)** получают выгоду от активов

### Роли участников

| Роль | Описание |
|------|----------|
| **Settlor** | Учредитель траста, передаёт активы |
| **Trustee** | Управляющий, действует в интересах бенефициаров |
| **Co-Trustee** | Со-управляющий, совместные полномочия |
| **Protector** | Надзорное лицо, контролирует trustee |
| **Beneficiary** | Получатель выгоды от траста |

### Типы трастов

- **Revocable (Отзывной)** — учредитель может изменить или отменить
- **Irrevocable (Безотзывной)** — не может быть изменён без согласия бенефициаров

## Функционал модуля

### 1. Управление трастами

- Создание и редактирование трастов
- Указание юрисдикции и типа
- Назначение trustee и protector
- Привязка документов (deed, amendments)

### 2. Бенефициары

- Добавление primary и contingent бенефициаров
- Указание долей (если фиксированные)
- Описание прав и ограничений
- История изменений через события

### 3. Распределения (Distributions)

- Создание запросов на распределение
- Указание суммы, валюты, цели
- Workflow одобрения (approval)
- Интеграция с модулем Liquidity
- Отслеживание статуса: draft → pending → approved → paid

### 4. События

Типы событий:
- **beneficiary_change** — изменение бенефициаров
- **trustee_change** — смена управляющего
- **amendment** — поправка к документам
- **distribution_decision** — решение о распределении
- **other** — прочие события

Каждое событие проходит согласование и фиксируется в audit trail.

### 5. Календарь и триггеры

- Даты ежегодных обзоров
- Даты истечения полномочий
- Напоминания о ключевых событиях
- Автоматические уведомления

### 6. Полномочия и governance

**Trust Powers:**
- Право на распределения
- Право изменения бенефициаров
- Инвестиционная дискреция
- Правила одобрения

**Approval Rules:**
- Пороги для одобрений
- Требуемые роли
- Количество одобряющих

## Документы

Ключевые документы траста:
- **Trust Deed** — учредительный документ
- **Amendments** — поправки
- **Minutes** — протоколы заседаний
- **Letters of Wishes** — пожелания учредителя
- **Distribution Letters** — письма о распределениях

Все документы хранятся в Document Vault с версионированием.

## Интеграция с другими модулями

| Модуль | Интеграция |
|--------|------------|
| **Document Vault** | Хранение deed, amendments, minutes |
| **Liquidity** | Распределения как obligations |
| **Net Worth** | Активы в трастах |
| **Workflow** | Approvals и задачи |
| **Partnerships** | Если траст владеет долями |

## Типовые сценарии

### Добавление нового бенефициара

1. Перейти в траст → вкладка Beneficiaries
2. Нажать "Add Beneficiary"
3. Заполнить форму (имя, тип, доля)
4. Система создаст событие **beneficiary_change**
5. Событие отправляется на approval (Trustee + Compliance)
6. После одобрения бенефициар активируется

### Создание распределения

1. Открыть траст → вкладка Distributions
2. Нажать "Create Distribution"
3. Выбрать бенефициара, указать сумму и цель
4. Прикрепить документы (authorization letter)
5. Submit на approval
6. После одобрения → Mark as Paid
7. Создаётся obligation в Liquidity

### Прикрепление документов

1. Открыть траст → вкладка Documents
2. Нажать "Attach Document"
3. Выбрать документ из Document Vault или загрузить новый
4. Указать тип (deed, amendment, etc.)
5. Документ привязывается к трасту

### Создание календарного триггера

1. Перейти на вкладку Calendar
2. Нажать "Add Trigger"
3. Указать название, дату, тип (review/expiration/reminder)
4. Система отправит уведомление за N дней

## Audit Trail

Все действия логируются:
- Создание/изменение траста
- Добавление/изменение бенефициаров
- Создание распределений и одобрения
- Прикрепление документов
- Изменения в calendar

## RBAC и доступ

| Роль | Права |
|------|-------|
| **Owner/Admin** | Полный доступ, одобрение изменений |
| **Trustee** | Создание распределений, событий |
| **Compliance** | Одобрение изменений бенефициаров |
| **Operations** | Прикрепление документов, задачи |
| **External Advisor** | Только чтение shared документов |
| **Client-safe** | Только опубликованные summaries |
