# Data Governance, Lineage и Why This Number

## Обзор

Модуль Data Governance обеспечивает enterprise-уровень доверия к данным и цифрам в системе Wealth OS. Он помогает ответить на ключевые вопросы:

- **Откуда эта цифра?** — Data Lineage показывает полный путь данных от источника до KPI
- **Почему это значение?** — Why This Number объясняет формулу, входные данные и допущения
- **Можно ли доверять?** — Trust Badge и Quality Score дают объективную оценку качества

## Ключевые возможности

### 1. Data Lineage
Визуализация пути данных:
- Источники: какие коллекции и поля используются
- Трансформации: какие вычисления производятся
- Результаты: какие метрики формируются

### 2. Why This Number
Для каждого KPI доступно объяснение:
- Определение метрики
- Формула расчета
- Входные данные со ссылками на источники
- Этапы обработки
- Допущения и ограничения

### 3. Data Quality Score
Оценка качества данных по 4 критериям:
- **Completeness** — полнота заполнения обязательных полей
- **Freshness** — актуальность данных (возраст asOf)
- **Consistency** — отсутствие конфликтов между источниками
- **Coverage** — количество и разнообразие источников

### 4. Reconciliation Views
Сверки между системами:
- IBOR vs ABOR — инвестиционная vs бухгалтерская книга
- Cash vs Bank — наличные vs банковская выписка
- Positions vs Custodian — позиции vs данные кастодиана
- GL vs Subledger — главная книга vs субсчета

### 5. Overrides
Управление корректировками:
- Ручные корректировки значений
- Реклассификации
- Исправления маппинга
- Workflow согласования (draft → pending → approved → applied)

### 6. Trust Badges
Индикаторы доверия:
- **Verified** — данные актуальны и качественны
- **Estimated** — данные оценочные или неполные
- **Stale** — данные устарели

## Интеграции

- **Exception Center (Module 48)** — при низком качестве или расхождении создаются исключения
- **MDM (Module 46)** — качество мастер-данных влияет на общий score

## AI Assist

AI-помощник может:
- Объяснить lineage простым языком
- Предложить возможные причины расхождений
- Обобщить риски качества данных

**Важно:** AI-пояснения носят информационный характер и требуют проверки.

## RBAC

- **Admin** — полный доступ
- **CFO/Controller** — просмотр, согласование overrides
- **Operations** — запуск quality checks, создание draft overrides
- **Compliance** — просмотр, согласование для sensitive данных
- **Client** — только Trust Badge (в клиентском портале)

## Дисклеймер

Пояснения носят информационный характер. Финальные выводы требуют проверки бухгалтером и ответственными лицами.
