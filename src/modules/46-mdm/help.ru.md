# MDM - Master Data Management

## Обзор

Модуль MDM (Master Data Management) обеспечивает единый справочник ключевых сущностей,
устранение дублей и формирование "золотой записи" (Golden Record).

## Ключевые возможности

### People 360
- Члены семьи
- Доверительные управляющие (trustees)
- Советники (advisors)
- Контакты провайдеров

### Entity Registry
- Домохозяйства (households)
- Холдинговые структуры
- Трасты и фонды
- Партнерства и SPV

### Account Registry
- Банковские счета
- Кастодиальные счета
- Брокерские счета
- Крипто-кошельки

### Asset Registry
- Тикеры и биржевые инструменты
- Фонды
- Частные сделки (private deals)
- Недвижимость
- Коллекционные объекты

## Identity Resolution

### Поиск дублей
- Fuzzy matching по имени, email, телефону
- Exact matching по идентификаторам (SSN, ISIN, LEI)
- Настраиваемые веса полей
- Пороговые значения совпадения

### Merge управление
- Сравнение side-by-side
- Выбор survivorship правил
- Ручной выбор значений
- Подтверждение ответственным лицом

## Data Stewardship

### Очередь исключений
- Missing sources - поля без источника
- Conflicting values - конфликтующие значения
- Low confidence - низкая уверенность
- Invalid format - неверный формат
- Stale data - устаревшие данные

### Назначение и resolution
- Назначение стюарду
- Отслеживание SLA
- Resolution notes
- Audit trail

## Source-first подход

Каждое поле имеет:
- **chosenValue** - выбранное значение (Golden Record)
- **sources** - список значений по источникам
- **confidence** - уровень уверенности (0-100%)
- **override** - флаг ручного override

## Survivorship правила

### Приоритет источников
1. Manual entry - ручной ввод
2. Custodian API - данные от кастодиана
3. Bank feed - банковские фиды
4. Bloomberg/Refinitiv - рыночные данные
5. Legacy import - исторический импорт

### Логика выбора
- Предпочитать непустые значения
- Предпочитать свежие данные
- Учитывать приоритет источника

## RBAC

| Роль | Доступ |
|------|--------|
| Admin | Полный доступ |
| CFO/Controller | Просмотр entities/accounts, ограниченные merges |
| Operations Analyst | Управление очередью, предложение merges |
| Compliance | Просмотр identity fields, утверждение merges |
| Client/Advisor | Нет доступа к MDM |

## Audit Trail

Записываются все действия:
- Создание/обновление записи
- Override поля
- Запуск matching
- Игнорирование дубликата
- Старт/подтверждение merge
- Создание/изменение правил

## Дисклеймер

> MDM рекомендации и совпадения носят информационный характер.
> Слияние записей требует подтверждения ответственным лицом.

## AI Assist

AI помощник может:
- Предложить потенциальные дубли
- Сформировать план merge
- Нормализовать адреса и имена

**Важно:** AI только предлагает, человек подтверждает (Human in the loop).
