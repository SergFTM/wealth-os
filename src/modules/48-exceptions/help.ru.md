# Центр исключений

## Обзор

Центр исключений — это единая операционная панель для управления всеми проблемами данных и процессов в системе Wealth OS. Модуль консолидирует исключения из всех источников: ошибки синхронизации, расхождения сверок, отсутствующие документы, устаревшие цены, нарушения SLA и просроченные согласования.

## Основные возможности

### Очередь исключений
- **Единая очередь** — все исключения в одном месте
- **Триаж** — назначение ответственных, установка приоритетов
- **SLA-контроль** — автоматическое отслеживание сроков
- **Связь с источником** — прямые ссылки на первичные объекты

### Кластеризация
- **Группировка похожих** — автоматическое объединение связанных исключений
- **Массовые действия** — применение решений ко всему кластеру
- **Выявление паттернов** — понимание системных проблем

### Правила автоматизации
- **Авто-назначение** — маршрутизация по типу и источнику
- **Авто-эскалация** — повышение приоритета при приближении SLA
- **Авто-закрытие** — закрытие при исправлении источника

### AI-ассистент
- **Суммаризация** — краткое описание проблемы
- **Предложение шагов** — рекомендации по устранению
- **Поиск похожих** — выявление связанных исключений

## Типы исключений

| Тип | Описание | Источник |
|-----|----------|----------|
| Sync | Ошибка синхронизации | Модуль интеграций (14) |
| Recon | Расхождение сверки | Модули GL/Positions (2/39/42) |
| Missing Doc | Отсутствует документ | Хранилище документов (5) |
| Stale Price | Устаревшая цена | Ценообразование (16) |
| Approval | Просрочено согласование | Согласования (7/28/44) |
| Vendor SLA | Нарушение SLA поставщика | Управление вендорами (43) |
| Security | Инцидент безопасности | Безопасность (17) |

## Статусы

- **Open** — новое исключение
- **Triage** — на рассмотрении
- **In Progress** — в работе
- **Closed** — закрыто

## Уровни критичности

- **OK** — информационное, низкий приоритет
- **Warning** — требует внимания
- **Critical** — срочное решение

## SLA-политики

Политики SLA определяют:
- Срок на решение по умолчанию
- Порог предупреждения о приближении дедлайна
- Время до эскалации
- Роли для уведомления

## Роли и доступ

| Роль | Доступ |
|------|--------|
| Operations Analyst | Полный триаж и управление |
| Compliance | Просмотр compliance-исключений |
| CIO/CFO | Просмотр сводки |
| Client/Advisor | Нет доступа |

## Аудит

Все действия логируются:
- Создание исключения
- Изменение назначения
- Изменение приоритета
- Обновление шагов устранения
- Запуск правил
- Закрытие/открытие

## Советы

1. Регулярно проверяйте исключения с риском SLA
2. Используйте кластеризацию для выявления системных проблем
3. Настройте правила для типичных сценариев
4. Проверяйте ссылки на источник перед закрытием

---

*Центр исключений предназначен для операционной команды. Решения требуют проверки человеком.*
